The **Send API** in LangGraph is a powerful feature that enables dynamic, data-driven fan-out within
a compiled graph. Unlike static edges that connect nodes in a predetermined manner, the `Send()`
function allows for conditional edge functions to return multiple `Send` objects at runtime. Each
`Send` object specifies a destination node and the state dictionary that should be sent to that
node.  ### How It Works - A `Send` object consists of:   - The name of the destination node.   - A
state dictionary containing the data to be sent. - When a conditional edge returns multiple `Send`
objects, LangGraph schedules all of them to be executed in parallel, facilitating efficient
processing.  ### Example Here’s a simple example of how the `Send` API can be used: ```python from
langgraph.types import Send  def dispatch_workers(state: MyState) -> list[Send]:     return
[Send("worker_node", {**state, "task": t}) for t in state["tasks"]] ```  ### Use Cases The Send API
is particularly useful for implementing map-reduce patterns, where: - A large problem is divided
into smaller, independent sub-problems (map). - Each sub-problem is solved in parallel. - The
results are aggregated (reduce).  Common applications include: - Parallel document retrieval. -
Multi-agent research pipelines. - Batch data processing within a single graph run.  ### Convergence
All branches created by the `Send()` API converge at a single "fan-in" node. LangGraph ensures that
it waits for all parallel branches to complete before invoking this convergence node, which receives
a fully merged state from all branches, making the parallel execution seamless for downstream logic.
This dynamic and parallel processing capability makes the Send API a valuable tool for optimizing
workflows in LangGraph.
